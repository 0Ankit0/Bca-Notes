Teaching outcomes

  • Deliver complete syllabus coverage with theory + practical integration.
  • Prepare students for assignments, term exam, lab exam, and viva.
  • Align lesson flow with the officially allocated 8 teaching hours.

3.1 Why frameworks matter

Frameworks provide reusable routing, template rendering, middleware integration, and security helpers. This reduces boilerplate and improves maintainability.

3.2 Flask setup and structure

Guide students through project structure: app entry file, templates folder, static assets, configuration, and dependency file.

3.3 Routes, views, templates

Teach URL-to-function mapping and Jinja template rendering. Introduce dynamic pages through variable interpolation and loops.

3.4 Request/response and form handling

Build GET/POST forms, validate input on server side, and return user-friendly feedback using message flashing.

3.5 Cookies, sessions, redirects, file uploads

Explain state handling limitations in HTTP and how Flask sessions maintain user context. Include secure file upload patterns.

3.6 Sending email in Flask

Cover SMTP basics, secure credential handling via environment variables, and simple use-cases (password reset/notifications).

Classroom pacing for 8 hours

  1. Flask intro and first app (2h)
  2. Template + form workshop (2h)
  3. Session/cookies/file upload practical (2h)
  4. Email integration and debugging (2h)
