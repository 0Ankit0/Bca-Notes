6.1 Web services and SOA

Service-oriented architecture decomposes systems into interoperable services communicating over standards-based interfaces.

6.2 SOAP model

Introduce envelope-based XML messaging, strict contracts, and enterprise integration contexts where SOAP still appears.

6.3 RESTful web services

Explain resource-oriented design, URI patterns, statelessness, and HTTP methods (GET, POST, PUT, PATCH, DELETE).

6.4 JSON vs XML

JSON is lightweight and frontend-friendly; XML is verbose but schema-rich. Teach when each format is practical.

6.5 Building and consuming APIs

Create a Flask REST API for student records and consume an external API with Python requests to reinforce integration skills.

Classroom pacing for 5 hours

  1. Concepts + protocol comparison (1.5h)
  2. Flask API build session (2h)
  3. External API consumption and response parsing (1.5h)
