1.1 Overview of Web-I and transition to Web-II

Web-I focused on client-side technologies. Web-II extends that knowledge to server-side execution, persistent storage, secure sessions, and deployment pipelines.

1.2 HTTP request and response message flow

Teach students to read the structure of HTTP requests (method, URI, headers, body) and responses (status code, headers, body), including common codes such as 200, 301, 400, 401, 404, and 500.

1.3 Client-side vs server-side technologies

Use classroom comparison examples: input validation in browser vs trusted validation in server, client rendering vs server rendering, and performance/security trade-offs.

1.4 Web clients and servers

Explain browsers and API clients as consumers. Demonstrate server roles using Apache, Nginx, and IIS as request handlers, static asset servers, and reverse proxies.

1.5 Server-side architecture

Monolithic architecture is easier to start with and debug in class projects, while micro-services improve scalability and team autonomy in larger systems.

1.6 Backend language ecosystem

Compare Python, PHP, and Java on learning curve, deployment style, framework ecosystems, and enterprise use cases.

1.7 MVC architecture

Model handles data and business rules, View handles rendering, Controller handles request orchestration. Map this to Flask route-view-model patterns.

Classroom pacing for 4 hours

  1. Concept + architecture diagrams (1.5h)
  2. Live HTTP inspection using browser dev tools (1h)
  3. Mini case study and MVC mapping activity (1.5h)
