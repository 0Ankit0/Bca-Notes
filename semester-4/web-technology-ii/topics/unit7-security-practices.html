Unit 7: Security in Web Applications

Detailed teaching notes • 8 hours

Home / Semester 4 / Web Technology II / Unit 7 Detailed Notes

Teaching outcomes

  • Deliver complete syllabus coverage with theory + practical integration.
  • Prepare students for assignments, term exam, lab exam, and viva.
  • Align lesson flow with the officially allocated 8 teaching hours.

7.1 Security foundations

Define assets, threats, vulnerabilities, and attack surfaces in web applications. Emphasize proactive defense mindset.

7.2 Core security principles

Least privilege, defense in depth, and fail-secure defaults should be embedded into every module of application design.

7.3 Common vulnerabilities

Cover SQL injection, XSS, CSRF, and DoS with live examples. Teach root causes and prevention patterns.

7.4 Best practices

Input validation/output encoding, secure authentication and authorization (MFA, RBAC), hashed passwords, secure session handling, logging, and secure uploads.

7.5 Transport + infrastructure security

Introduce SSL/TLS, HTTPS enforcement, firewalls, proxies, VPNs, and host/network/application threat mitigation.

7.6 Secure design guidelines

Provide checklist-based design review before deployment.

Classroom pacing for 8 hours

  1. Fundamentals and principles (2h)
  2. Vulnerability demonstrations (3h)
  3. Mitigation coding lab (2h)
  4. Security review and checklist exercise (1h)
