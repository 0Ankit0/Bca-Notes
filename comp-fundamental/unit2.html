Unit 2: Basic Organization of Computer

Exam Focus: This unit is critical for understanding how a computer works internally. Expect detailed questions on the functional units and the block diagram of a computer system.

2.1 Basic Functions of Computer

At its core, a computer performs five fundamental operations to process data and provide meaningful information:

  • Input: Receiving data and instructions from the user or other sources.
  • Processing: Performing arithmetic and logical operations on the data as per instructions.
  • Output: Presenting the processed results to the user.
  • Storage: Saving data and instructions for immediate or future use.
  • Control: Managing and coordinating all the operations within the computer system.

2.2 Basic Functional Organization of Computer

A digital computer system is organized into several interconnected functional units that work together to execute tasks. This organization is often represented by a block diagram.

Exam Question Alert: You will very likely be asked to draw and explain the block diagram of a digital computer. Practice drawing it and understanding the role of each unit.

Functional Units Explained:

2.2.1 Input Unit

The Input Unit is responsible for accepting data and instructions from the outside world (users or other systems) and converting them into a binary format that the computer can understand. Without an input unit, a computer would be an isolated system, unable to receive any commands or data.

  • Examples: Keyboard, Mouse, Scanner, Microphone, Joystick, Webcams.

2.2.2 Central Processing Unit (CPU)

Often called the "brain" of the computer, the CPU is responsible for executing instructions, performing calculations, and managing the overall flow of data. It fetches instructions from memory, decodes them, and then executes them.

  • Arithmetic Logic Unit (ALU): This component performs all arithmetic operations (addition, subtraction, multiplication, division) and logical operations (comparisons like greater than, less than, equal to). It is where the actual computation happens.
  • Control Unit (CU): The CU is the manager of the computer system. It directs and coordinates the entire computer system, fetching instructions from memory, interpreting them, and issuing control signals to other units to ensure proper execution. It doesn't perform actual data processing but controls all operations.
  • Registers: These are small, high-speed storage locations within the CPU itself. They temporarily hold data and instructions that the CPU is currently processing, providing very fast access to critical information.

2.2.3 Memory Unit

The Memory Unit is where data and instructions are stored. It acts as the computer's short-term and long-term recall system.

  • Primary Memory (Main Memory): This is fast memory directly accessible by the CPU. It holds data and programs that are currently in use. It is typically volatile (data is lost when power is off).
    • RAM (Random Access Memory): Volatile, read/write memory.
    • ROM (Read Only Memory): Non-volatile, stores startup instructions (BIOS).
  • Secondary Storage: This provides permanent storage for large amounts of data and programs. It is non-volatile.
    • Examples: Hard Disk Drives (HDD), Solid State Drives (SSD), USB drives, Optical Disks.

2.2.4 Output Unit

The Output Unit is responsible for presenting the processed information (results) from the computer to the user in a human-readable or usable format. It converts binary data back into a form we can understand.

  • Examples: Monitor, Printer, Speakers, Projector.

2.2.5 Bus System

The Bus System is a set of electronic pathways that connect all the internal components of a computer to the CPU and main memory. It facilitates the transfer of data, addresses, and control signals between these components.

  • Address Bus: Carries memory addresses from the CPU to other components, indicating where data should be read from or written to.
  • Data Bus: Carries the actual data being transferred between the CPU, memory, and other peripherals.
  • Control Bus: Carries control signals from the CPU to other components to manage and synchronize operations (e.g., read/write signals, timing signals).

2.3 The System Concept

The system concept in computing refers to the idea that all the functional units (input, processing, output, storage, and control) are not isolated but work together in an integrated and coordinated manner to achieve a common goal: processing information. Each component plays a vital role, and the failure of one can affect the entire system's performance. This holistic view emphasizes the interdependencies and interactions between hardware and software components to form a complete, functional computer system.
