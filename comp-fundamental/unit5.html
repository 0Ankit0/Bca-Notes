Unit 5: Storage Devices

Exam Focus: This unit is highly important for understanding how data is stored and retrieved. Expect detailed questions on memory hierarchy, differences between various memory types (RAM vs. ROM, HDD vs. SSD), and the characteristics of primary and secondary storage.

5.1 Introduction to Storage Devices

Storage devices are fundamental components of any computer system, responsible for permanently or temporarily holding data, programs, and processing results. They are essential for a computer to operate, retain information, and execute tasks. Without effective storage, a computer would lose all its data every time it's turned off.

5.2 Memory Hierarchy

Exam Question Alert: Explaining the memory hierarchy is a very common question. Understand the levels, their characteristics, and why this hierarchy exists.

The memory hierarchy in a computer system is a structured arrangement of storage devices, organized by speed, cost, and capacity. The goal is to provide the CPU with fast access to data while keeping the overall cost of memory low. Faster, more expensive, and smaller memory (like registers and cache) are closer to the CPU, while slower, cheaper, and larger memory (like secondary storage) are further away.

  • Registers: Smallest, fastest, most expensive. Located inside the CPU.
  • Cache Memory: Small, very fast memory between CPU and main memory.
  • Primary Memory (RAM): Main working memory, fast, volatile.
  • Secondary Storage: Large capacity, non-volatile, slower.

5.3 Registers

Registers are the smallest, fastest, and most expensive memory components in the computer system. They are tiny storage areas located directly inside the CPU.

  • Function: Temporarily hold instructions, memory addresses, and data being actively manipulated by the ALU and Control Unit during the CPU's fetch-decode-execute cycle. They are crucial for rapid instruction execution.

5.4 Cache Memory

Exam Question Alert: The use and function of Cache Memory are often tested. Understand how it speeds up CPU access.

Cache Memory is an extremely fast, small memory buffer that sits between the CPU and Primary Memory (RAM). Its purpose is to speed up data access for the CPU.

  • Function: It stores copies of data and instructions that the CPU is likely to use next. When the CPU needs data, it first checks the cache. If found (cache hit), it's retrieved quickly. If not (cache miss), the CPU retrieves it from RAM and stores a copy in the cache for future use.
  • Levels: Cache is typically divided into L1 (internal to CPU, fastest), L2 (external to CPU, fast), and sometimes L3 (shared among CPU cores).

5.5 Primary Memory (Main Memory)

Exam Question Alert: Differentiating between RAM and ROM, and primary vs. secondary memory, are frequent exam questions.

Primary Memory, also known as Main Memory, is the working memory of the computer. The CPU must have programs and data loaded into Primary Memory before it can process them.

Characteristics of Primary Memory:

  • Volatile: Data is lost when the power is turned off (e.g., RAM).
  • Direct Access: The CPU can access any location quickly and directly.
  • High Speed: Fast enough to keep up with the CPU's processing demands.
  • Limited Capacity: Smaller storage capacity compared to secondary storage.

Types of Primary Memory:

  • RAM (Random Access Memory):
    • Volatile, read/write memory.
    • Holds the operating system, application programs, and data currently in use.
    • DRAM (Dynamic RAM): Most common type, needs constant refreshing, slower, cheaper.
    • SRAM (Static RAM): Faster, more expensive, used for cache memory, does not need refreshing.
  • ROM (Read-Only Memory):
    • Non-volatile, usually read-only.
    • Stores crucial firmware and instructions (like the BIOS) needed to start the computer.
    • PROM (Programmable ROM): Blank chips that can be programmed once.
    • EPROM (Erasable Programmable ROM): Can be erased using UV light and reprogrammed.
    • EEPROM (Electrically Erasable Programmable ROM): Can be erased and rewritten electrically (used in modern flash BIOS chips).

Difference between RAM and ROM:

Feature RAM ROM
Volatility Volatile (data lost without power) Non-volatile (data retained without power)
Usage Working memory, temporary storage Stores start-up instructions (firmware)
Operation Read and Write Primarily Read

Difference between DRAM and SRAM:

Feature DRAM (Dynamic RAM) SRAM (Static RAM)
Speed Slower Faster (used for Cache memory)
Density High (more memory in less space) Low (less memory in more space)
Cost Less Expensive More Expensive
Refresh Requires constant electrical refresh Does not require constant refresh

5.6 Secondary Memory (Auxiliary Storage)

Exam Question Alert: Expect questions on the characteristics and types of secondary memory, and detailed comparisons like HDD vs. SSD.

Secondary Memory, also known as Auxiliary Storage, provides a permanent, long-term storage solution for all programs and data. It is non-volatile, meaning data persists even when the computer is turned off.

Characteristics of Secondary Memory:

  • Non-Volatile: Data is permanent until intentionally erased.
  • High Capacity: Designed to store vast amounts of data.
  • Lower Speed: Much slower access time compared to Primary Memory.
  • Economical: Lower cost per bit of storage than Primary Memory.

Types of Secondary Memory:

  • Magnetic Storage: Uses magnetized material to store data. Data is read/written by tiny electromagnets.
    • Examples: Hard Disk Drives (HDD), Magnetic Tapes.
  • Optical Storage: Uses lasers to read and write data pits and lands on a reflective surface.
    • Examples: CD-ROM, DVD-ROM, Blu-ray Discs.
  • Solid State Storage: Uses flash memory (EEPROM) technology, containing no moving parts.
    • Examples: Solid State Drives (SSD), USB Flash Drives, SD Cards.

Difference between Hard Disk Drive (HDD) and Solid State Drive (SSD):

Feature Hard Disk Drive (HDD) Solid State Drive (SSD)
Technology Magnetic spinning platters Flash memory chips
Speed Slower (mechanical movement) Much Faster (electronic access)
Durability Sensitive to shocks and movement Highly durable (no moving parts)
Noise Audible spinning noise Silent

Direct vs. Sequential Access Magnetic Memory:

  • Direct Access (Random Access): Allows the device to jump directly to any location on the storage medium to retrieve data. Access time is relatively independent of data location (e.g., HDDs, SSDs).
  • Sequential Access: Data must be read or written in the order it was recorded. To access data in the middle, all preceding data must be read first (e.g., Magnetic Tapes).
